# USB-UPS monitoring via NUT

Optional Telegraf input that exposes a USB HID UPS as metrics:

```text
USB UPS → NUT usbhid-ups → NUT upsd (127.0.0.1:3493) → Telegraf [[inputs.upsd]]
```

It is **off by default** and fully gated by `telegraf_ups_enable`. When disabled the
role installs nothing UPS-related and the rendered `telegraf.conf` is byte-for-byte
unchanged. This path is independent of `telegraf_apcupsd_enable` (a different daemon).

## Supported platforms

Debian 12, Debian 13, and current compatible Ubuntu (24.04+). The role auto-detects the
NUT service model at runtime:

- **NUT 2.8** (Debian 12/13, Ubuntu 24.04) — driver enumerator: each driver runs as
  `nut-driver@<name>.service`, regenerated by `nut-driver-enumerator.service`.
- **NUT 2.7.x** (e.g. Ubuntu 22.04) — the driver is started by `nut-server` itself.

`nut-monitor.service` (upsmon) is **masked** unless shutdown is explicitly enabled.

## Telegraf version

`[[inputs.upsd]]` requires **Telegraf ≥ 1.24**. The role's default install path
(`state: latest` from the InfluxData repo) satisfies this. If you pin an older build via
`telegraf_deb_url`, ensure it is ≥ 1.24 — the role does **not** fall back to `inputs.exec`.

## Variables

| Variable | Default | Purpose |
| --- | --- | --- |
| `telegraf_ups_enable` | `false` | Master switch for the whole feature. |
| `telegraf_ups_packages` | `[nut-server, nut-client]` | Distro packages to install. |
| `telegraf_ups_name` | `"ups"` | NUT section / host name (`[A-Za-z0-9._-]+`). |
| `telegraf_ups_description` | `"USB UPS"` | `desc` in `ups.conf`. |
| `telegraf_ups_driver` | `"usbhid-ups"` | NUT driver. |
| `telegraf_ups_driver_port` | `"auto"` | `port` in `ups.conf`. |
| `telegraf_ups_driver_vendorid` | `""` | USB `vendorid` (rendered only if set). |
| `telegraf_ups_driver_productid` | `""` | USB `productid` (rendered only if set). |
| `telegraf_ups_driver_serial` | `""` | USB `serial` (rendered only if set). |
| `telegraf_ups_driver_options` | `{}` | Extra `ups.conf` driver options (mapping). |
| `telegraf_ups_listen_address` | `"127.0.0.1"` | `upsd` LISTEN address + Telegraf `server`. |
| `telegraf_ups_listen_port` | `3493` | `upsd` LISTEN port + Telegraf `port`. |
| `telegraf_ups_interval` | `"10s"` | Telegraf input interval. |
| `telegraf_ups_force_float` | `true` | Telegraf `force_float`. |
| `telegraf_ups_stringify_ids` | `true` | Telegraf `stringify_ids`. |
| `telegraf_ups_username` | `""` | Optional upsd user (rendered only if set). |
| `telegraf_ups_password` | `""` | Secret — set via Vault, never commit. |
| `telegraf_ups_additional_fields` | `["*"]` | NUT vars to collect; `"*"` = everything. |
| `telegraf_ups_shutdown_*` | see defaults | Optional host shutdown (off by default). |

Leave `vendorid`/`productid`/`serial` empty to auto-detect a single attached USB UPS
(`driver = usbhid-ups`, `port = auto`). Anonymous read works without a username/password.

### Which metrics are collected

By default **every** variable the UPS exposes through `upsd` is collected — the default
`telegraf_ups_additional_fields: ["*"]` tells Telegraf's `upsd` input to add all
variables on top of its standard mapped set. There is nothing to enumerate by hand.
Set a narrower glob list (e.g. `["battery.*", "ups.status"]`) only if you deliberately
want fewer metrics.

## First-install note: USB permissions

A USB UPS that is already plugged in when NUT is installed keeps `root:root` device-node
ownership, so the `nut`-user driver would fail with *"insufficient permissions"*. The role
handles this automatically: when the NUT packages are installed it reloads udev rules and
re-triggers USB events (`udevadm control --reload-rules` + `udevadm trigger`) so the device
is re-tagged for group `nut`, then restarts the driver. No replug or reboot is needed.

## ⚠️ Automatic shutdown is OFF by default

With `telegraf_ups_shutdown_enable: false` (default) the role keeps `nut-monitor`
**stopped, disabled and masked** and ships no shutdown-capable `upsmon.conf`. NUT is used
purely as a telemetry source — no `SHUTDOWNCMD`, no kill-power, no poweroff.

Enabling `telegraf_ups_shutdown_enable: true` lets NUT power the host off on battery/low
events. **A single-board computer (e.g. RockPi) will not power back on by itself when mains
return** — an automatic shutdown can leave it dead until someone re-plugs it. Enable only
deliberately; a username and password are then required.

## Minimal example

A single USB UPS is auto-detected (`port = auto`) and every variable is collected:

```yaml
telegraf_ups_enable: true
telegraf_ups_name: "myups"
telegraf_ups_description: "USB UPS"
```

## Extended example

Disambiguate a specific device and tune the driver (all values are placeholders):

```yaml
telegraf_ups_enable: true
telegraf_ups_name: "myups"
telegraf_ups_description: "USB UPS"
telegraf_ups_driver_vendorid: "XXXX"      # from `lsusb -nn`
telegraf_ups_driver_productid: "YYYY"
telegraf_ups_driver_serial: "SERIAL123"
telegraf_ups_driver_options:
  pollinterval: 5
# Restrict the collected set (optional; default ["*"] collects everything):
telegraf_ups_additional_fields:
  - "battery.*"
  - "ups.status"
```

## Manual verification (on the target)

Replace `<ups-name>` with `telegraf_ups_name`:

```bash
lsusb -nn
upsc -l 127.0.0.1:3493
upsc <ups-name>@127.0.0.1:3493
systemctl status nut-server
systemctl status nut-driver@<ups-name>     # NUT 2.8 driver instance
systemctl status nut-monitor               # expected: masked/inactive (shutdown off)
systemctl status telegraf
journalctl -u nut-server
journalctl -u telegraf
```
